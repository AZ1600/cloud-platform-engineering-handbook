# Backstage Developer Portal

> An internal developer platform that centralises the software catalogue, ownership, technical documentation, governance, and self-service workflows.

**Repository:** <https://github.com/AZ1600/platform-engineering-backstage-portal>  
**Last verified:** 2026-07-11 against the repository README

## Why It Matters

Platform engineering is not only about infrastructure. A platform must make approved capabilities easy for developers to discover and use. Backstage provides a common interface over services, documentation, templates, and integrations.

## Verified Capabilities

- Backstage software catalogue and service ownership.
- TechDocs-oriented documentation management.
- GitHub and Kubernetes integration described by the project.
- Catalogue overview, dependency graph, and service entity evidence.
- TypeScript, Node.js, Docker, Backstage, and Kubernetes stack.

## Current and Planned Scope

The README shows the catalogue and portal foundation. Software templates, scaffolder workflows, expanded Kubernetes management, CI/CD, observability, cost views, and internal APIs are listed as future improvements and must not be presented as completed features.
